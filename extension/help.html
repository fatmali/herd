Guide

Everything you need to know about organizing your tabs with Herd.

How It Works

Herd organizes your browser tabs into tab groups — the colored, collapsible labels you see in Edge or Chrome. It works in three steps:

  1. Scan — Looks at all open tabs across all windows.
  2. Classify — Matches each tab's URL against your rules. First match wins.
  3. Group — Places tabs into named, colored groups. Unmatched tabs stay ungrouped.
💡 Herd never closes tabs or changes their order. It only assigns group labels and colors.

Rules & Categories

A rule (category) is a named group with:

  • Name — the tab group title (e.g. "Code Review")
  • URL patterns — if a tab's URL matches any pattern, it belongs here
  • Color — the tab group color in the browser

Priority Order

Rules are evaluated top-to-bottom. The first match wins — once matched, no further rules are checked.

⚠️ If you have overlapping patterns (e.g. *github.com* and github.com/*/pull/*), place the more specific rule first.

URL Pattern Syntax

Patterns use simple wildcard syntax. * matches any number of characters (including none).

How Matching Works

  1. The https:// or http:// prefix is stripped from both URL and pattern
  2. Comparison is case-insensitive
  3. Each * expands to match anything

Examples

PatternMatchesDoesn't Match
*github.com*Any GitHub page—
github.com/*/pull/*GitHub PRs onlyIssues, repos, profiles
*figma.com*Any Figma page—
localhost:*Any localhost port127.0.0.1
docs.*docs.microsoft.comlearn.microsoft.com/docs
*myproject*Any URL with "myproject"—
💡 When in doubt, wrap with * on both sides: *keyword*. Matches the keyword anywhere in the URL.

Common Patterns

# Match a specific domain
*example.com*

# Match a specific path
github.com/myorg/myrepo*

# Match a subdomain
*.staging.myapp.com*

# Match multiple ports
localhost:3000*
localhost:8080*

Auto-Organize & Scheduling

Herd can run automatically. Configure in Settings → Schedule.

Auto-organize toggle (popup)
Master on/off. When off, only "Organize Now" triggers grouping.
Schedule interval (settings)
15 min, 30 min, 1 hour, 2 hours, or manual-only.
Collapse inactive groups
Minimizes inactive groups to reduce noise.

When Does It Run?

  • On install — immediately organizes existing tabs
  • On schedule — at the configured interval
  • On demand — via "Organize Now" in the popup
  • After rule changes — when you assign a tab via right-click

Right-Click Menu

Quickly assign any page to a category:

  1. Right-click anywhere on the page
  2. Hover "Herd: Add tab to category"
  3. Pick a category — or "+ New category..."

This adds the tab's domain as a pattern. For example, right-clicking linear.app/myteam/issue/ENG-123 adds *linear.app*.

💡 This is the fastest way to teach Herd. Browse normally, spot an ungrouped tab, right-click → assign.

Tips & Best Practices

Start broad, refine later

Begin with *github.com*. As tabs get miscategorized, split into github.com/*/pull/* for PRs and github.com/*/issues* for issues.

Use few categories

5–8 is the sweet spot. Too many defeats the purpose. Let unmatched tabs stay ungrouped.

Let the AI agent focus for you

With the AI bridge connected, ask your agent to "focus on auth migration" — it will create a temporary group for related tabs.

Export your rules

Settings → Export. Share with teammates or keep a backup before experimenting.

Multiple windows

Each window is organized independently using the same rules.

When connected to an AI agent

If you enable the AI bridge, your agent gains access to the same data Herd itself uses — tab titles, URLs, and rules. Specifically:

  • The bridge runs on 127.0.0.1:9922 only — never exposed to the network
  • All requests require a bearer token from ~/.herd/auth.json (user-only file permissions)
  • Tab data is held in memory only while the bridge runs — never written to disk
  • Error logs (~/.herd/bridge.log) contain timestamps and error messages only, never tab data
⚠️ Tab URLs may reveal sensitive info — internal tool URLs, ticket numbers, document titles. Consider this when connecting AI agents that send data to external servers.
💡 You can disconnect the AI bridge at any time with npx herd-tabs --uninstall. The extension continues working standalone with zero external communication.

Connecting to AI

Herd works standalone, but you can also control it from any AI assistant — GitHub Copilot, Claude Code, ChatGPT, or any agent that supports skill files.

Setup (one-time, ~2 minutes)

  1. Install the bridge — open a terminal and run:
npx herd-tabs --install

This registers a native messaging host so the bridge starts automatically whenever your browser is open. No manual process to manage.

  1. Restart your browser (close all windows, reopen)
  2. Copy the skill file to your AI agent:
AgentCopy shared/skill.md to
GitHub Copilot CLI~/.copilot/skills/herd.md
Claude Code~/.claude/skills/herd.md
ChatGPTPaste contents into custom instructions
Other agentsWherever your agent reads skill/prompt files

What you can say

Once linked, just talk naturally:

  • "Organize my tabs"
  • "Focus my tabs on the auth migration"
  • "Add a rule for Figma tabs in the Design category"
  • "What tabs do I have open?"
  • "Remove the incidents category"

How it works

  1. Browser opens → extension spawns the bridge via native messaging
  2. Bridge runs an HTTP server on localhost:9922 (token-authenticated)
  3. Your AI agent reads the skill file, which tells it about the available endpoints
  4. Agent calls the endpoints (using curl/fetch) → bridge → extension → tabs grouped
🔒 All API requests require a bearer token stored in ~/.herd/auth.json. The server only binds to localhost — it's never exposed to the network.

Verifying it works

Check bridge status:

npx herd-tabs --status

Or ask your agent: "What's my herd status?" — if it responds with your tab count, you're connected.

Uninstalling

npx herd-tabs --uninstall

Removes the native host, registry entries, and auth token. The extension continues working standalone.
