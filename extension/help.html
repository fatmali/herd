Guide

Everything you need to know about organizing your tabs with Herd.

How It Works

Herd organizes your browser tabs into tab groups — the colored, collapsible labels you see in Edge or Chrome. It works in three steps:

  1. Scan — Looks at all open tabs across all windows.
  2. Classify — Matches each tab's URL against your rules. First match wins.
  3. Group — Places tabs into named, colored groups. Unmatched tabs stay ungrouped.
💡 Herd never closes tabs or changes their order. It only assigns group labels and colors.

Rules & Categories

A rule (category) is a named group with:

  • Name — the tab group title (e.g. "Code Review")
  • URL patterns — if a tab's URL matches any pattern, it belongs here
  • Color — the tab group color in the browser

Priority Order

Rules are evaluated top-to-bottom. The first match wins — once matched, no further rules are checked.

⚠️ If you have overlapping patterns (e.g. *github.com* and github.com/*/pull/*), place the more specific rule first.

URL Pattern Syntax

Patterns use simple wildcard syntax. * matches any number of characters (including none).

How Matching Works

  1. The https:// or http:// prefix is stripped from both URL and pattern
  2. Comparison is case-insensitive
  3. Each * expands to match anything

Examples

PatternMatchesDoesn't Match
*github.com*Any GitHub page—
github.com/*/pull/*GitHub PRs onlyIssues, repos, profiles
*figma.com*Any Figma page—
localhost:*Any localhost port127.0.0.1
docs.*docs.microsoft.comlearn.microsoft.com/docs
*myproject*Any URL with "myproject"—
💡 When in doubt, wrap with * on both sides: *keyword*. Matches the keyword anywhere in the URL.

Common Patterns

# Match a specific domain
*example.com*

# Match a specific path
github.com/myorg/myrepo*

# Match a subdomain
*.staging.myapp.com*

# Match multiple ports
localhost:3000*
localhost:8080*

Focus Topics

Focus topics highlight tabs related to your current work. Matching tabs are pulled into a special "Current Focus" group (yellow) at the top.

How It Works

Topics match against both the tab title and URL as a case-insensitive substring.

# Focus topic: "auth-migration"
# ✓ Title: "PR #4421 - Migrate auth-migration to MSAL"
# ✓ URL:   "github.com/org/repo/issues?q=auth-migration"

Setting Focus Topics

  • Popup — Type comma-separated topics in the Focus Topics field
  • CLI — The herd service can set topics via WorkIQ based on your calendar
💡 Use specific keywords: ticket numbers (PROJ-1234), branch names (auth-migration), or codenames. Avoid generic words like "meeting" that match too broadly.

Focus vs. Rules

Focus topics take priority over rules. A matching tab goes to "Current Focus" even if it also matches a category. Clear your topics to return tabs to normal groups.

Auto-Organize & Scheduling

Herd can run automatically. Configure in Settings → Schedule.

Auto-organize toggle (popup)
Master on/off. When off, only "Organize Now" triggers grouping.
Schedule interval (settings)
15 min, 30 min, 1 hour, 2 hours, or manual-only.
Collapse inactive groups
Minimizes groups not matching your current focus to reduce noise.

When Does It Run?

  • On install — immediately organizes existing tabs
  • On schedule — at the configured interval
  • On demand — via "Organize Now" in the popup
  • After rule changes — when you assign a tab via right-click

Right-Click Menu

Quickly assign any page to a category:

  1. Right-click anywhere on the page
  2. Hover "Herd: Add tab to category"
  3. Pick a category — or "+ New category..."

This adds the tab's domain as a pattern. For example, right-clicking linear.app/myteam/issue/ENG-123 adds *linear.app*.

💡 This is the fastest way to teach Herd. Browse normally, spot an ungrouped tab, right-click → assign.

Tips & Best Practices

Start broad, refine later

Begin with *github.com*. As tabs get miscategorized, split into github.com/*/pull/* for PRs and github.com/*/issues* for issues.

Use few categories

5–8 is the sweet spot. Too many defeats the purpose. Let unmatched tabs stay ungrouped.

Focus topics for deep work

Type your topic in the popup before a focus session. Related tabs surface; everything else collapses.

Export your rules

Settings → Export. Share with teammates or keep a backup before experimenting.

Multiple windows

Each window is organized independently using the same rules.

Connecting to AI

Herd works great on its own — but you can also control it from any AI assistant. The extension exposes a local API and an MCP server that any agent can use.

How it works

  1. The Herd service runs locally and bridges your AI agent ↔ the extension
  2. Your agent sends commands (organize, set focus, add rules) via MCP or HTTP
  3. The extension picks up commands and applies them to your browser

Setup for Claude Code / Claude Desktop

Add to your claude_desktop_config.json or .mcp.json:

{
  "mcpServers": {
    "herd": {
      "command": "npx",
      "args": ["herd-tabs"]
    }
  }
}

Setup for GitHub Copilot CLI

Add to your ~/.copilot/config.json under MCP servers:

{
  "mcpServers": {
    "herd": {
      "command": "npx",
      "args": ["herd-tabs"]
    }
  }
}

Then just say "organize my tabs" or "focus my tabs on auth-migration" in conversation.

Setup for any other agent (HTTP API)

If your AI tool can make HTTP calls, point it at the local API:

# List tabs
GET http://localhost:9922/tabs

# Organize tabs
POST http://localhost:9922/organize
Body: { "focus_topics": ["auth-migration"] }

# Set focus
POST http://localhost:9922/focus
Body: { "topics": ["sprint-review"] }

# Check status
GET http://localhost:9922/status

Available MCP tools

ToolDescription
herd_organizeOrganize all tabs into groups

💡 The service starts automatically when your agent invokes it via npx herd-tabs. No need to run it manually.

Verifying the connection

Once the service is running, visit http://localhost:9922/health — you should see {"status":"ok"}. The extension popup will also show a "Connected" indicator when the service is reachable.
